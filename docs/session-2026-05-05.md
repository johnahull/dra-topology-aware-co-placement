### Key finding: launcher image override
KubeVirt v1.8.2 ignores `VIRT_LAUNCHER_IMAGE` env var. The only way to use a custom launcher is to tag the custom image as the stock tag (`quay.io/kubevirt/virt-launcher:v1.8.2`) in containerd's image store:
```bash
sudo ctr -n k8s.io images rm quay.io/kubevirt/virt-launcher:v1.8.2
sudo ctr -n k8s.io images tag localhost/virt-launcher:dra-multi-numa-v3 quay.io/kubevirt/virt-launcher:v1.8.2
```

### VMI spec for DRA GPU passthrough
GPUs must be explicitly referenced in `spec.domain.devices.gpus` with `claimName` (the resourceClaim ref name) and `requestName` (the expanded sub-request name):
```yaml
devices:
  gpus:
  - name: gpu0
    claimName: quarter        # matches spec.resourceClaims[].name
    requestName: partition-vfio-gpu-nvidia-com  # webhook-expanded sub-request name
```

### Recovery procedure after force-delete or hung state:
```bash
sudo rm -f /var/lib/kubelet/dra_manager_state
sudo rm -f /var/lib/kubelet/plugins/gpu.nvidia.com/checkpoint.json
sudo rm -f /var/lib/kubelet/plugins/gpu.nvidia.com/pu.lock
kubectl get pods -n nvidia-dra -o name | grep kubelet | xargs kubectl delete -n nvidia-dra --force --grace-period=0
sudo systemctl restart kubelet
```
