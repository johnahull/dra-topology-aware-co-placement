# Session Notes — 2026-05-05

## Topology Coordinator Changes

### Branch: `test/all-fixes-combined` in `k8s-dra-topology-coordinator`

#### 1. NUMA attribute aliases
- Changed `AttrNUMANode` from `dra.net/numaNode` to `resource.kubernetes.io/numaNode`
- Added `numaNodeAliases` list for all known driver-specific NUMA names
- Coordinator now recognizes any driver's NUMA attribute

#### 2. Socket attribute aliases
- Added `socketAliases` list: `resource.kubernetes.io/cpuSocketID`, `dra.cpu/cpuSocketID`, etc.
- Required for half-partition creation (by socket)

#### 3. Partition hierarchy — successive bisection
- Walk socket → NUMA → PCIe root boundaries
- Assign half/quarter/eighth to boundaries that actually split
- Skip boundaries that don't further divide
- XE8640: half=2 (socket), quarter=4 (PCIe root), full=1
- HGX B200: half=2 (socket), quarter=4 (NUMA), eighth=8 (PCIe root), full=1

#### 4. CEL `has()` guard
- CEL selectors use `has()` to prevent runtime errors on devices missing NUMA attribute (e.g. tunl0)

#### 5. DeviceClass hash stability (3 fixes)
- Key rawSlices/DevicesByDriver by slice name (not driver/pool) — handles multiple slices with same pool
- Remove device counts from profile string — prevents hash changes from transient device count changes
- Populate model from informer store after cache sync — prevents partial profiles during startup

#### 6. Shared capacity for proportional partitions
- Devices with count < subdivisions that publish consumable capacity use capacity mode
- CPU and memory share via DRAConsumableCapacity, GPUs remain exclusive

#### 7. DeviceClass override in topology rules
- New `deviceClass` field in ConfigMap topology rules
- Maps `gpu.nvidia.com` driver → `vfio.gpu.nvidia.com` DeviceClass for VFIO GPUs
- ConfigMap deployed on XE8640 (removed from R760xa since its GPUs use nvidia driver)

## NVIDIA GPU Driver — All-VFIO Workarounds (XE8640)

All 4 H100 SXM5 GPUs pre-bound to vfio-pci via `vfio-pci.ids=10de:2330` kernel cmdline.

### Issues encountered:
1. **Init container** — `nvidia-smi` fails when all GPUs are VFIO-bound. Fixed by patching DS init to `exit 0`
2. **compute-domains container** — NVML fails without nvidia-managed GPUs. Fixed by replacing command with `sleep` loop
3. **Stale checkpoint locks** — hung VFIO unbind leaves `/var/lib/kubelet/plugins/gpu.nvidia.com/pu.lock`, blocking all prepare/unprepare. Fix: delete lock file + restart GPU plugin
4. **Stale DRA manager state** — force-deleted claims leave stale UIDs in kubelet memory. Fix: delete `/var/lib/kubelet/dra_manager_state` + restart kubelet
5. **Phantom gpu-vfio-2** — GPU driver publishes both `type=gpu` and `type=vfio` for same physical GPU when nvidia driver is loaded. Fixed by pre-binding all GPUs to vfio-pci

### Recovery procedure after force-delete or hung state:
```bash
# On the XE8640:
sudo rm -f /var/lib/kubelet/dra_manager_state
sudo rm -f /var/lib/kubelet/plugins/gpu.nvidia.com/checkpoint.json
sudo rm -f /var/lib/kubelet/plugins/gpu.nvidia.com/pu.lock
kubectl delete pod -n nvidia-dra -l app.kubernetes.io/name=nvidia-dra-driver-gpu --force --grace-period=0
sudo systemctl restart kubelet
```
